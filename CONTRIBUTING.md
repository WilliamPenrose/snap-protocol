# Contributing to SNAP

Thanks for your interest in contributing to SNAP! This document explains how to participate.

## Ways to Contribute

### Feedback on the Spec

The most valuable contribution right now is feedback on the specification:

- **Open an issue** for questions, suggestions, or concerns
- **Start a discussion** for broader topics
- Be specific â€” point to the exact section and explain your thinking

### Pull Requests

For changes to the spec:

1. **Fork** the repository
2. **Create a branch** for your changes
3. **Make your edits** following the style guide below
4. **Open a PR** with a clear description

## Style Guide

### Writing Style

- Use plain, direct language
- Prefer short sentences
- Lead with the most important information
- Use examples liberally

### Code Examples

- Use JSON for data structures
- Include realistic values, not lorem ipsum
- Add comments only when necessary

**Good**:
```json
{
  "identity": "bc1p5d7rjq7g6rdk2yhzqnt9dp8wvscrplqk0z...",
  "name": "Code Assistant"
}
```

**Avoid**:
```json
{
  "identity": "<YOUR_IDENTITY_HERE>",
  "name": "<AGENT_NAME>"
}
```

### Document Structure

- Start with a one-sentence summary
- Use tables for field definitions
- Put complete examples at the end
- Link to related documents

## Questions?

Open an issue or start a discussion. We're happy to help!
